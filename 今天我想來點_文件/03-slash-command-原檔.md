# 任務：重建「斜線四門 hover 抬起」區段

請依以下規格做一個 full viewport 的 hero 區段。**所有座標都根據 1834×1062 參考畫布等比換算成 % / vw。**

## 視覺結構（5 層，從後到前）

1. **背景**：水泥質感圖（路徑 `/cement-light.png`，找不到就用 `#c8c8c8` 暖灰底）
2. **4 個漢字區塊**：每個區塊用 `clip-path` 切成「兩條斜線之間的梯形」，內含
   - 該區塊的水泥背景（會跟著抬起的部分）
   - 兩行漢字（top / bot）
   - 對應的編號（01–04）
3. **5 條白色斜線**：用 SVG 畫，永遠在最上層之一
4. **左側文字區**：TODAY / SOMETHING MORE… / 今天我想來點… / body 兩段
5. （數字已搬到第 2 層內）

## 必要規格

### Hover 行為（重點）

每個區塊（`.char-pair.pN`）滑鼠移上去要同時做：
1. `transform: translateY(-10px)` — 抬起
2. `filter: brightness(1.13) drop-shadow(0 14px 22px rgba(0,0,0,0.4))` — 變亮 + 陰影
3. `z-index: 6` — 浮到最上層
4. `clip-path` 擴寬 ±6%（每個四角各往外移 6 個百分點）→ 露出完整漢字
5. 該區段的 01/02/03/04 編號要**跟著抬**（所以數字 DOM 要寫在 `.char-pair` 裡面）
6. 全部變化用 `transition: ... 0.4s cubic-bezier(0.16, 1, 0.3, 1)`

## 細節要注意

- `.num` 必須 `pointer-events: none`，否則會擋住下層斜線區塊的 hover
- 每組 `.char-pair` 是 `inset: 0` 全頁面，內部用絕對定位放文字 — 這樣 clip-path 才能直接吃 viewport %
- clip-path 過渡要在 transition 裡加 `clip-path`，否則 hover 時邊界會「閃」
- 5 條斜線層級在區塊之上、但 hover 區塊抬起後（z-6）會蓋過線 — 這是預期效果
