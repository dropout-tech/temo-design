# Four Doors 斜線區段 — 技術文件

> 對應檔案：[public/four-doors-prototype.html](../public/four-doors-prototype.html)
> 對應指令：`/diagonal-doors`

## 這是什麼

TEMO 官網的「今天我想來點」服務選擇區段。視覺重點：

- 整面水泥牆當底
- 5 條白色斜線把畫面切成 4 個梯形區段
- 每個區段裡有 2 行漢字（如「品牌 / 平面」）+ 對應編號（01–04）
- 滑鼠 hover 到任一區段 → 那塊板子抬起來、變亮、長出陰影、變寬、字完整露出

## 視覺架構（5 層）

```
z-5  ┌─────────────────────────────────┐
     │ 左側白字（TODAY、SOMETHING MORE）│
z-3  │ 5 條白色斜線（SVG）              │
z-2  │ 4 個 .char-pair（斜線區段）       │  ← hover 時這層升到 z-6
     │   ├─ .char-panel-bg（水泥）       │
     │   ├─ .char-text（漢字 2 行）      │
     │   └─ .num（01/02/03/04）          │
z-1  │ 整面背景（水泥圖）                │
     └─────────────────────────────────┘
```

關鍵：**每個 `.char-pair` 是全頁面 `inset: 0` 的容器**，用 `clip-path` 切出梯形區段才看得到內容。這樣 clip-path 的座標可以直接用 viewport %（不用算相對位置）。

## clip-path 怎麼算出來的

5 條斜線在參考圖（1834×1062）的座標：

```
L1: (260, 983)  → (901, 170)
L2: (533, 976)  → (1175, 161)
L3: (806, 967)  → (1447, 154)
L4: (1065, 966) → (1706, 153)
L5: (1292, 1027)→ (1832, 339)
```

把每條線**延伸到 y=0 跟 y=100%**，就能拿到 4 個梯形（兩條相鄰線中間就是一個區段）。

### 延伸後的 x 座標

| 線 | x 在 y=0 | x 在 y=100% |
|---|---|---|
| L1 | 56.44% | 10.78% |
| L2 | 70.98% | 25.37% |
| L3 | 85.52% | 39.86% |
| L4 | 99.60% | 53.94% |
| L5 | 114.40% | 68.95% |

### 4 個區段的預設 clip-path（順時針：TL → TR → BR → BL）

```css
.char-pair.p1 { clip-path: polygon(56.44% 0, 70.98% 0, 25.37% 100%, 10.78% 100%); }
.char-pair.p2 { clip-path: polygon(70.98% 0, 85.52% 0, 39.86% 100%, 25.37% 100%); }
.char-pair.p3 { clip-path: polygon(85.52% 0, 99.60% 0, 53.94% 100%, 39.86% 100%); }
.char-pair.p4 { clip-path: polygon(99.60% 0, 114.40% 0, 68.95% 100%, 53.94% 100%); }
```

p4 右上點 `114.40%` 超過視窗，是正常的 — clip-path 多邊形可以延伸到元素外，CSS 只會取元素內的交集當作可見區。

### Hover 時擴寬 ±6%

每個多邊形的 4 個頂點：
- 左邊 2 個（TL、BL）→ x 各 **-6%**
- 右邊 2 個（TR、BR）→ x 各 **+6%**

```css
.char-pair.p1:hover { clip-path: polygon(50.44% 0, 76.98% 0, 31.37% 100%, 4.78% 100%); }
.char-pair.p2:hover { clip-path: polygon(64.98% 0, 91.52% 0, 45.86% 100%, 19.37% 100%); }
.char-pair.p3:hover { clip-path: polygon(79.52% 0, 105.60% 0, 59.94% 100%, 33.86% 100%); }
.char-pair.p4:hover { clip-path: polygon(93.60% 0, 120.40% 0, 74.95% 100%, 47.94% 100%); }
```

調整這個數字可以控制「hover 時露出多少」：
- 想再寬一點 → +7 或 +8
- 想再窄一點 → +4 或 +5

## Hover 效果的 5 件事

```css
.char-pair {
  transition: transform 0.4s cubic-bezier(0.16, 1, 0.3, 1),
              filter 0.4s ease,
              clip-path 0.4s cubic-bezier(0.16, 1, 0.3, 1);
}
.char-pair:hover {
  transform: translateY(-10px);                                /* ① 抬 */
  filter: brightness(1.13) drop-shadow(0 14px 22px rgba(0,0,0,0.4));  /* ② 亮 + ③ 影 */
  z-index: 6;                                                  /* ④ 浮頂 */
  clip-path: polygon(... 寬版 ...);                             /* ⑤ 變寬 */
}
```

⑥（隱形效果）：數字 `<div class="num nN">` 直接寫在 `.char-pair` 裡面，所以會跟著 translateY 一起抬起來。

## 容易踩的坑

1. **`.num` 一定要 `pointer-events: none`** — 不然 01–04 會擋掉下層 `.char-pair` 的 hover
2. **transition 要包含 `clip-path`** — 不然 hover 邊界會「閃」一下、沒過渡
3. **數字 DOM 要寫在 `.char-pair` 裡** — 否則 hover 時數字不會跟著抬
4. **clip-path 多邊形頂點數要一致** — 預設跟 hover 都 4 點才能動畫
5. **每組 `.char-pair` 是 `inset: 0` 全頁面**（不是只佔斜線內的小區塊）— 這是 clip-path 能用 viewport % 的前提
